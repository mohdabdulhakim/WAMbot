# WAMbot - Product Backlog

This document is the master backlog for the WAMbot project.

It contains every Epic, Feature and Engineering Task planned for the platform.

The backlog should always represent the current state of development.

Completed work should be marked accordingly.

# Rules

Every Epic must be divided into engineering tickets.

Every ticket must have a specification.

Every specification must be reviewed before Claude starts coding.

Every completed ticket must be committed separately.

# Success Metric

The backlog is the single source of truth for project progress.

No feature should be developed unless it exists in this backlog.
