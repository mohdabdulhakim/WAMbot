# AI Project Rules

Version: 1.0

These rules apply to every coding task generated by Claude.

---

# Primary Goal

Write production-ready code for WAMbot.

Every implementation must be scalable, maintainable, secure, and modular.

---

# General Rules

Never overwrite existing files without permission.

Never delete code unless instructed.

Never remove functionality.

Never introduce breaking changes.

Always preserve backward compatibility.

---

# Architecture Rules

Follow existing folder structure.

Respect module boundaries.

Keep business logic inside services.

Controllers should remain thin.

Database access must use Prisma.

Environment variables must never be hardcoded.

---

# Code Quality

Write readable code.

Follow TypeScript best practices.

Avoid duplicated logic.

Use dependency injection.

Keep functions small.

Meaningful variable names only.

---

# Security

Validate every input.

Sanitize user data.

Protect sensitive routes.

Hash passwords.

Never expose secrets.

Implement proper authentication.

---

# Database

Every schema change requires:

Prisma schema update

Migration

Seeder update if needed

Documentation update

---

# API

Use REST conventions.

Consistent response format.

Proper HTTP status codes.

Meaningful error messages.

Version APIs where appropriate.

---

# Frontend

Responsive UI.

Accessible components.

Reusable components.

Avoid inline styles.

Use loading states.

Handle errors gracefully.

---

# Git Rules

One feature per commit.

Small commits.

Meaningful commit messages.

Never commit secrets.

---

# Testing

Every module should include:

Unit tests

Validation tests

API tests where applicable

---

# Documentation

Whenever architecture changes:

Update documentation.

Whenever API changes:

Update API documentation.

Whenever database changes:

Update database documentation.

---

# Performance

Optimize queries.

Avoid unnecessary API calls.

Implement pagination.

Implement caching where useful.

Lazy load heavy components.

---

# AI Development Workflow

Before writing code:

Understand requirements.

Review related documents.

Review existing implementation.

Identify dependencies.

Plan implementation.

Write code.

Review code.

Suggest improvements.

---

# Communication

Explain major architectural decisions.

Highlight risks.

Mention assumptions.

Suggest better approaches if applicable.

---

# Golden Rule

Always think like a senior software engineer building a commercial SaaS product, not a coding assistant generating isolated files.
